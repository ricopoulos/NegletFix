## Corrected diagnosis

The Meta Horizon error *"only the owner can do it"* refers to the **headset's primary Meta account**, NOT the developer organization owner.

- Confirmed by Meta community thread 1322016 (Quest 3, same symptom, all verifications green): *"It CAN NOT turn developer mode on or off. It has to be done from the primary account."*
- Inviting daughter to Eric's dev org as Admin (the April recommendation) is likely insufficient by itself. The toggle has to be flipped from **her** Meta Horizon app on **her** phone, with **her** account satisfying all the developer prerequisites.

## Hard requirements for whoever flips the toggle

That account (= the headset's primary account) must:
1. Be a registered Meta developer (developers.meta.com)
2. Be a member of a verified developer organization (own or joined)
3. Have org verification complete — 2FA OR credit card on file
4. **Be at least 18 years old**
5. Run latest Meta Horizon app + latest Quest firmware

## The branch point — daughter's account status

Outcome depends entirely on her age + account type:

- **18+, independent Meta account** → she creates her own dev account + org, verifies, toggles dev mode from her Meta Horizon app. Simplest path. Eric's existing org is irrelevant.
- **13–17, teen account** → cannot register as Meta developer. Only paths: (a) factory-reset Quest and re-pair to Eric's account; or (b) wait until she's 18.
- **10–12, parent-managed** → same as teen account; factory-reset is the only route.
- **Also check**: parental supervision can explicitly **Block developer mode** via Family Center — if that toggle is on, dev mode is hidden regardless of org setup.

App Sharing on Quest preserves purchases for secondary profiles even if primary changes — worth noting before any factory reset.
